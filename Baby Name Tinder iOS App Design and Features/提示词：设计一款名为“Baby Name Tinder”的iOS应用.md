# 提示词：设计一款名为“Baby Name Tinder”的iOS应用

## 1. 应用核心概念

请为我设计一款创新的iOS应用，主题是“Baby Name Tinder”。这款应用的核心目标是帮助整个家庭以一种有趣、协作的方式为新生儿挑选名字。应用的核心交互模式应类似于Tinder，通过左滑（不喜欢）和右滑（喜欢）来筛选名字，并最终找到所有家庭成员都喜欢的名字。

## 2. 核心功能需求

### a. 用户与协作
- **家庭圈 (Family Circle):** 用户可以创建一个家庭圈，邀请家人（如配偶、祖父母等）加入。所有圈内成员都可以参与选名过程。
- **独立筛选:** 每个家庭成员都可以独立地对名字进行“喜欢”或“不喜欢”的操作。
- **匹配列表 (Match List):** 当一个名字被家庭圈内的所有成员都“喜欢”时，该名字会自动进入“匹配列表”。这是最终的备选名字清单。

### b. 名字数据库与筛选
- **多源数据库:** 名字库应支持从不同来源进行选择，例如：
  - **地域/文化:** 中文、英文、日文、法文等。
  - **风格:** 现代、古典、神话、自然等。
- **详细信息:** 每个名字都应提供丰富的背景信息，包括：
  - **含义与寓意:** 名字的详细解释和象征意义。
  - **来源:** 名字的起源故事或文化背景。
  - **发音:** 提供音频发音（如果可能）。
  - **相关数据:** 如流行度趋势、名人使用等。
- **高级筛选:** 用户可以根据以下条件筛选名字：
  - **性别:** 男、女、中性。
  - **首字母/尾字母。**
  - **长度或音节数。**

### c. 核心交互
- **滑动选名:** 应用主界面应为卡片堆叠形式，每张卡片展示一个名字及其基本信息。用户可以左滑表示“不喜欢”，右滑表示“喜欢”。
- **收藏夹 (Favorites):** 除了“喜欢”，用户还可以将特别中意的名字加入个人收藏夹，以便后续回顾和讨论。
- **分享功能:** 用户可以将单个名字或整个“匹配列表”分享到社交媒体或通讯应用中。

## 3. 页面设计 (UI/UX)

请使用 **Tailwind CSS 和 HTML** 为以下关键页面生成可视化原型：

1.  **启动与家庭圈创建页面:** 简洁的应用介绍，引导用户创建或加入一个家庭圈。
2.  **主滑动选名页面:** 核心交互界面，以卡片形式展示名字，包含清晰的“喜欢”和“不喜欢”按钮，并能快速查看名字基本含义。
3.  **名字详情页面:** 点击卡片后进入，展示名字的完整信息（含义、来源、发音、流行度等）。
4.  **匹配列表页面:** 展示所有家庭成员都喜欢的名字列表，突出显示“全家都同意！”的喜悦感。
5.  **个人中心/设置页面:** 包括个人收藏夹、筛选器设置、家庭圈管理和订阅状态。

## 4. 收费模式：免费 + 高级订阅 (Freemium)

请设计一个合理的收费模式，平衡免费功能和付费价值。

### a. 免费版功能
- 每天可滑动有限数量的名字（例如50个）。
- 创建或加入一个家庭圈（最多2名成员）。
- 查看名字的基本含义。
- 无限访问“匹配列表”。

### b. 高级订阅 (Premium) 功能
- **无限滑动:** 无限制地浏览所有名字。
- **高级家庭圈:** 可邀请更多家庭成员加入（例如最多10人）。
- **高级筛选器:** 解锁所有筛选条件（如按风格、来源、音节等筛选）。
- **查看完整名字详情:** 获取名字的全部信息，包括流行度趋势、历史渊源等深度内容。
- **无广告体验。**

请将以上所有元素整合为一个完整、详细、可执行的应用设计方案。
